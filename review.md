# Laravel Lesson レビュー①

## Todo一覧機能

### Todoモデルのallメソッドで実行しているSQLは何か
→　SELECT * FROM todos;

### Todoモデルのallメソッドの返り値は何か
→　todosテーブルの全データを格納したコレクションオブジェクトを返す

### 配列の代わりにCollectionクラスを使用するメリットは
→　動的なサイズ変更が可能、ループ処理を抽象化した強力なメソッドを利用できる、安全な操作が可能、コードの可読性が向上する

### view関数の第1・第2引数の指定と何をしているか
→　第1引数はファイルの指定、第2引数は渡したいデータを記載することで連想配列の形で渡すことができる

### index.blade.phpの$todos・$todoに代入されているものは何か
$todos　→　todosテーブルの全データを格納したCollectionオブジェクト
$todo　→　$todosの全データから該当のデータを1行ずつ順番に取り出している

## Todo作成機能

### Requestクラスのallメソッドは何をしているか
→　フォームから送信された入力欄を連想配列で取得できる

### fillメソッドは何をしているか
→　連想配列で取得した値をモデルの各カラムへ一括代入できる

### $fillableは何のために設定しているか
→　便利な一括代入には脆弱性があり、対策をせずに使用すると悪意のあるユーザに攻撃されてしまう恐れがあるため

### saveメソッドで実行しているSQLは何か
→　INSERT INTO todos (content) VALUES ('入力された値');

### redirect()->route()は何をしているか
→　名前付きルートを指定して、ユーザーを一覧表示ページへ自動的に転送するメソッド

## その他

### テーブル構成をマイグレーションファイルで管理するメリット
→　SQLを知らなくても、PHPコードでテーブル操作ができる
　 Gitで共有することで、開発者全員が同じテーブルを作成することができる

### マイグレーションファイルのup()、down()は何のコマンドを実行した時に呼び出されるのか
up　→　php artisan migrate
down　→　php artisan migrate:rollback

### Seederクラスの役割は何か
→　DBにダミーデータを自動で作成できる

### route関数の引数・返り値・使用するメリット
→　引数：ルート名
　 返り値：ルートに対応するURLを生成
　 メリット：URLを変更しても、コード内のルート名を変えずに済むため、リンク切れなどのミスを防ぎ、修正の手間を減らせる

### @extends・@section・@yieldの関係性とbladeを分割するメリット
関係性：共通レイアウトを定義し、ページごとのコンテンツを埋め込むための親子関係を持っている
メリット：子ページが@extendsで大元のテンプレートを継承し、親の@yieldで指定した場所へ、子ページの@section(中身)を格納するため、保守性が高く、コードも中身だけを記載するため、ミスを防ぐことが可能

### @csrfは何のための記述か
→　偽造リクエストによる不正操作から守るCSRF対策のため

### {{ }}とは何の省略系か
→　XSS対策としてのエスケープ処理と変数や関数の返り値を省略したもの
